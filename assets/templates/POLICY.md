# Shared Context Policy

## Default Branch Policy

- Prefer branch-first updates.
- Direct pushes to the default branch require explicit team approval.

## Update Quality

- Only record meaningful facts, decisions, blockers, or handoff-critical changes.
- Keep facts and hypotheses clearly separated.

## Review Expectations

- Validate the shared context structure before committing.
- Review diffs before pushing.

## Conflict Handling

- Stop on divergence or conflicts.
- Reconcile manually instead of overwriting shared context.
